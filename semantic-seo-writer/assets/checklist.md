# SEO Content Writing Checklist — All 71 Rules

Quick-reference checklist for reviewing content against all rules. Check off each rule as you verify compliance.

---

## Content Optimization (Rules 1-20)

- [ ] **Rule 1:** Featured snippet answers are 40 words or fewer
- [ ] **Rule 2:** Boolean questions start with "Yes" or "No"
- [ ] **Rule 3:** Type/listing questions are structured with clear formatting
- [ ] **Rule 4:** No sentences start with "if"; minimal use of "also"; statements before conditions
- [ ] **Rule 5:** No analogies — all comparisons replaced with direct factual statements
- [ ] **Rule 6:** No back-references ("As stated before", "As explained in section Y")
- [ ] **Rule 7:** Every pronoun clearly refers to a specific entity (no coreference errors)
- [ ] **Rule 8:** No everyday/conversational language — formal and professional diction only
- [ ] **Rule 9:** No outbound links to citations or sources
- [ ] **Rule 10:** No product promotion in informational articles
- [ ] **Rule 11:** Semantic triples used (Subject + Special Fact + Main Information)
- [ ] **Rule 12:** No unnecessary sentences — related info combined where possible
- [ ] **Rule 13:** All statements are certain and definite (no possibility language)
- [ ] **Rule 14:** Specific counts and categories used (not vague descriptors)
- [ ] **Rule 15:** Correct verb context ("increase" for health, "improve" for skills, "develop" for growth)
- [ ] **Rule 16:** Authoritative sources cited before statements
- [ ] **Rule 17:** Declarations are consistent across all webpages
- [ ] **Rule 18:** Consistent style and tone across all documents
- [ ] **Rule 19:** Same part of speech used at the start of every list item
- [ ] **Rule 20:** Content is as short as possible but as long as necessary

## Semantic SEO & Structure (Rules 21-45)

- [ ] **Rule 21:** Boolean questions answered with "Yes/No" + supporting evidence
- [ ] **Rule 22:** Context vector flows logically from H1 to last heading
- [ ] **Rule 23:** All contextless/filler words removed
- [ ] **Rule 24:** Discourse integration optimized (clear topic sentences, transitions, consistent terms)
- [ ] **Rule 25:** Perspective richness added where applicable (diverse audience perspectives)
- [ ] **Rule 26:** Relevant entities and attribute values included for the macro context
- [ ] **Rule 27:** Focus on entities and their attributes, not just keywords
- [ ] **Rule 28:** No entity stuffing — attributes prioritized by prominence, relatedness, popularity
- [ ] **Rule 29:** Evidence expanded with statistical, anecdotal, scientific, and expert sources
- [ ] **Rule 30:** All sentences are factual, objective, verifiable, and direct
- [ ] **Rule 31:** Content is genuinely original (not just rephrased)
- [ ] **Rule 32:** Semantic SEO framework principles applied
- [ ] **Rule 33:** Examples given after every plural noun
- [ ] **Rule 34:** Sentences structured as Subject + Predicate + Object (semantic triples)
- [ ] **Rule 35:** Content types maintained (main, supplementary, ads) with proper balance
- [ ] **Rule 36:** Key terms connect to title, H1, and topical map
- [ ] **Rule 37:** Long-form answers use definitions, entity signifiers, and qualifiers
- [ ] **Rule 38:** Context not broken across paragraphs — one topic per paragraph
- [ ] **Rule 39:** Information graph follows logical order with simple transitions
- [ ] **Rule 40:** Anchor text matches target page title
- [ ] **Rule 41:** Tenses and modal verbs match between questions and answers
- [ ] **Rule 42:** Studies cited with year, organization, and findings
- [ ] **Rule 43:** Maximum information density per webpage, section, paragraph, and sentence
- [ ] **Rule 44:** Simple sentence structures (S+V+O) with direct language
- [ ] **Rule 45:** First sentence under heading matches heading format

## Advanced Techniques (Rules 46-71)

- [ ] **Rule 46:** Correct parts of speech usage (nouns, verbs, adjectives, adverbs)
- [ ] **Rule 47:** Perspective richness added for headings searchable by different user types
- [ ] **Rule 48:** "If" conditions placed in second part of sentence
- [ ] **Rule 49:** Attributes and contexts prioritized by interrogative term signals
- [ ] **Rule 50:** Safe, comprehensive answers with multiple evidence types
- [ ] **Rule 51:** Contextless words removed (basically, essentially, actually)
- [ ] **Rule 52:** Relevance configured by restructuring entity descriptions per query
- [ ] **Rule 53:** Same n-grams used at document start and end
- [ ] **Rule 54:** Ontology and taxonomy principles applied to entity organization
- [ ] **Rule 55:** Short-form questions preferred; long-form split into H2/H3 hierarchy
- [ ] **Rule 56:** Answer part bolded, not the search term
- [ ] **Rule 57:** Single macro context maintained per webpage
- [ ] **Rule 58:** Topic covered completely with every relevant detail
- [ ] **Rule 59:** Tables introduced with context explanation
- [ ] **Rule 60:** Answer immediately follows the question (no delay)
- [ ] **Rule 61:** Headings converted to question format where applicable
- [ ] **Rule 62:** Truth ranges provided when data varies across sources
- [ ] **Rule 63:** Unique n-grams and phrase combinations used
- [ ] **Rule 64:** Abbreviation in parentheses on first mention of any term
- [ ] **Rule 65:** Multiple measurement unit systems included
- [ ] **Rule 66:** Numeric values used instead of vague quantifiers
- [ ] **Rule 67:** Ordered and unordered lists used for structured content
- [ ] **Rule 68:** Short sentences used (one piece of information per sentence)
- [ ] **Rule 69:** Dependency trees kept short for clarity
- [ ] **Rule 70:** Research and university studies cited with specifics
- [ ] **Rule 71:** Vocabulary richness demonstrated with domain-specific terms and headword clusters
