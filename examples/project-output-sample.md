## Content Optimization Rules — Precision at the Sentence Level

Content Optimization, covering Rules 1 through 20, enforces precision in sentence structure, citation format, and content tone.

### Featured Snippet Optimization

Rule 1 requires answers to remain within a 40-word limit to trigger featured snippets. Short, direct answers increase the probability of appearing in Google's Position Zero. Rule 2 mandates that boolean answers start with "Yes" or "No" before providing supporting details.

### Semantic Triple Construction

Rule 11 mandates the use of semantic triples, which structure information as Subject + Special Fact + Main Information. Rule 34 reinforces this structure through the Subject-Predicate-Object framework.

The following YAML block demonstrates how semantic triples apply within the `SKILL.md` frontmatter:

```yaml
---
name: hasnain-seo-content-writing
description: >-
  Comprehensive SEO content writing and optimization skill based on semantic SEO
  principles. Enforces 71 rules covering featured snippets, entity optimization,
  sentence structure, content originality, factual writing, truth ranges, and
  vocabulary richness.
license: Proprietary
metadata:
  author: Hasnain Ayaz
  author_url: https://hasnainayaz.com/
  version: "1.0"
  framework: "Koray Tugberk GUBUR Semantic SEO"
---
```

The YAML frontmatter above establishes the skill identity. The `name` field must be lowercase with hyphens, matching the parent directory name exactly. The `description` field must be 1-1,024 characters and should include specific keywords that help agents identify relevant tasks.

### Citation and Authority Rules

Rule 16 requires citing authoritative sources before making factual claims. The format is: "According to [Source], [Statement]." Rule 42 specifies study citations using the format: "A [Year] study at [Organization] found that [Fact]." Rule 9 prohibits outbound links to citations to conserve link juice.

## Semantic SEO and Structure — Entity-Driven Content Architecture

Semantic SEO and Structure, covering Rules 21 through 45, governs content architecture through context vectors, entity focus, and discourse integration.

### Context Vectors and Heading Flow

Rule 22 requires a logical context vector from the first heading to the last heading. The context vector ensures that search engines interpret the entire document as a single, coherent topic. Rule 36 requires key terms to appear in both the title and H1 heading for clear topical focus.

### Entity-Attribute Prioritization

Rule 27 shifts the focus from keywords to entities, requiring content writers to identify and associate relevant entities with detailed attributes and facts. Rule 28 prevents entity stuffing by defining 3 prioritization criteria:

- **Attribute Prominence**: attributes without which the entity cannot be defined or exist
- **Attribute Relatedness**: attributes that connect to the source context and monetization strategy
- **Attribute Popularity**: attributes with measurable search demand in the targeted geographic region

### Tense and Modality Matching

Rule 41 enforces tense consistency between headings and their supporting paragraphs. A heading using present indefinite tense requires an answer in present indefinite tense. A heading containing a modal verb, such as "can" or "could," requires the supporting text to contain the same modal verb.

According to Google Patent No. US9213748B1, the system generates related questions based on search queries and ranks them by search frequency, answer quality, and source credibility.

## Advanced Techniques — Vocabulary Richness and Truth Ranges

Advanced Techniques, covering Rules 46 through 71, distinguish expert-level SEO content through truth ranges, unique n-grams, and domain-specific vocabulary.

### Truth Ranges for Data Consistency

Rule 62 introduces truth ranges. Data varies across sources. The content must provide minimum-to-maximum value ranges instead of single values. Truth ranges handle inconsistencies and demonstrate comprehensive research depth to search engine algorithms.

### Vocabulary Richness and N-Gram Profiles

Rule 71 requires vocabulary richness. There are 4 techniques for building a strong vocabulary profile:

1. Use unique, knowledge-domain terms instead of generic synonyms
2. Apply headword clusters that group related terms under a primary concept
3. Provide 3 examples for every plural noun mentioned in the text
4. Use "or" to introduce synonym groups that expand semantic coverage

Rule 63 mandates unique n-gram combinations. "Portable semantic SEO skill for AI coding agents" carries stronger topical signals than the generic phrase "SEO writing tool."

The directory structure above spans 1,615 lines across 5 files. The `SKILL.md` file contains the YAML frontmatter and condensed instructions. The `references/` directory stores detailed explanations and examples that agents load on demand. The `assets/` directory stores the 71-rule checklist for content review workflows.

## Progressive Disclosure — Efficient Context Window Management

The <a href="https://agentskills.io" target="_blank">Agent Skills specification</a> implements progressive disclosure through **3 loading stages**:

1. **Metadata stage (~100 tokens)**: the `name` and `description` fields load at startup for all installed skills
2. **Instructions stage (<5,000 tokens recommended)**: the full `SKILL.md` body loads when the agent activates the skill
3. **Resources stage (as needed)**: files in `references/`, `scripts/`, and `assets/` load only when the agent requires detailed examples or checklists

This 3-stage architecture prevents context window overflow. The main `SKILL.md` file remains under 500 lines, following the specification recommendation. Detailed reference material resides in separate files. The agent loads reference files only when a specific rule category requires elaboration.

The following `bash` command verifies the line count of each file in the skill directory:

```bash
wc -l SKILL.md references/*.md assets/*.md
```

The command above outputs the line count for each file and the total. The expected output for this skill is 1,615 total lines across 5 files.

## Deployment Across AI Platforms

The skill deploys to AI platforms by copying the skill directory to the platform-specific skills location. There are 3 primary deployment paths:

1. **Copy to `.agents/skills/`** for VS Code with GitHub Copilot integration
2. **Copy to `.claude/skills/`** for Claude Code environments
3. **Copy to the designated skills directory** for Cursor and other Agent Skills-compatible tools

The following `bash` command copies the skill directory to a VS Code Copilot project:

```bash
cp -r hasnain-seo-content-writing/ /path/to/project/.agents/skills/hasnain-seo-content-writing/
```

The command above creates a complete copy of the skill directory within the project. The agent discovers and activates the skill automatically when a content writing task matches the skill description.

### Whitelabeling the Skill for Custom Branding

The skill supports whitelabeling through 4 modifications in the `SKILL.md` frontmatter:

1. Update the `name` field to the new brand identifier (e.g., `hasnain-seo-content-writing`)
2. Update the `author` metadata field to the new author name (e.g., "Hasnain Ayaz")
3. Add the `author_url` metadata field pointing to the author website (e.g., `https://hasnainayaz.com/`)
4. Rename the parent directory to match the new `name` field exactly

The directory name must match the `name` field in the YAML frontmatter. A mismatch causes validation failures during skill discovery.
