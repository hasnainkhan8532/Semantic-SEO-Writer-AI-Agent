## Why Do AI Agents Need a Dedicated SEO Writing Framework?

AI agents need a dedicated SEO writing framework because default language models lack awareness of 3 critical ranking factors: entity-attribute relationships, dependency tree length, and semantic triple construction.

According to Google's Natural Language Processing (NLP) documentation, search algorithms evaluate content through Named Entity Recognition (NER), which identifies entities and their contextual attributes within a document. Without explicit rules, AI-generated content introduces 4 common ranking problems:

- Coreference errors where pronouns reference ambiguous entities
- Dependency trees exceeding optimal length due to compound sentence structures
- Entity stuffing that mimics keyword stuffing patterns
- Inconsistent tense and modality between question headings and answer paragraphs

The SEO content writing skill eliminates these 4 problems by enforcing explicit constraints at the sentence, paragraph, and document levels.

## What Are the 3 Core Categories of SEO Writing Rules?

The 71 rules divide into **3 core categories**: Content Optimization, Semantic SEO and Structure, and Advanced Techniques. Each category addresses a distinct layer of content quality that search engines evaluate during indexing and ranking.

### How Does Content Optimization Improve Rankings?

Content Optimization, covering Rules 1 through 20, improves rankings by enforcing precision in sentence structure, citation authority, and formatting consistency.

Rule 1 requires answers to remain within a 40-word limit to trigger featured snippets. Rule 11 mandates the use of semantic triples, which structure information as Subject + Special Fact + Main Information. Rule 16 requires citing authoritative sources, including research organizations, industry experts, and government agencies, before making factual claims.

There are 5 critical rules within this category that directly affect featured snippet eligibility:

1. Limit answers to 40 words per response
2. Start boolean answers with "Yes" or "No" before elaborating
3. Structure sentences using semantic triples
4. Cite authoritative sources before stating facts
5. Use the same part of speech at the start of every list item

According to a 2023 analysis at Ahrefs, pages with concise, directly structured answers capture featured snippets at a rate of 31% compared to 8% for pages with verbose, unstructured responses.

### How Does Semantic SEO Structure Content for Search Engines?

Semantic SEO and Structure, covering Rules 21 through 45, structures content for search engines by enforcing context vectors, discourse integration, and entity-attribute prioritization.

Rule 22 requires a logical context vector from the first heading to the last heading. This context vector ensures that search engines interpret the document as a single, coherent topic. Rule 27 shifts the focus from keywords to entities, requiring content writers to identify and associate relevant entities with their attributes and facts within a specific context.

There are 3 prioritization criteria for entity attributes defined by Rule 28:

- **Attribute Prominence**: attributes without which the entity cannot be defined
- **Attribute Relatedness**: attributes that connect to the source context and monetization strategy
- **Attribute Popularity**: attributes with measurable search demand in the targeted region

Rule 41 enforces tense and modality matching between question headings and answer paragraphs. A heading using present indefinite tense requires an answer in present indefinite tense. A heading containing a modal verb, such as "can" or "could," requires the answer to contain the same modal verb.

### What Advanced Techniques Distinguish Expert-Level SEO Content?

Advanced Techniques, covering Rules 46 through 71, distinguish expert-level SEO content through truth ranges, unique n-grams, and vocabulary richness.

Rule 62 introduces truth ranges. When data varies across sources, the content must provide minimum-to-maximum value ranges instead of single values. According to Koray Tugberk Gubur, truth ranges handle data inconsistencies and demonstrate comprehensive research depth to search engine algorithms.

Rule 71 requires vocabulary richness. Domain-specific words, headword clusters, and unique n-gram profiles signal genuine expertise to semantic engines. There are 4 techniques for building vocabulary richness:

1. Use unique, knowledge-domain terms instead of generic synonyms
2. Apply headword clusters that group related terms under a primary concept
3. Provide 3 examples for every plural noun mentioned in the text
4. Use "or" to introduce synonym groups that expand semantic coverage

## How Is the Skill Structured Using the Agent Skills Specification?

The skill is structured using the <a href="https://agentskills.io/specification" target="_blank">Agent Skills specification</a>, an open format for portable AI agent instructions. The specification requires a specific directory structure with 3 mandatory elements: a SKILL.md file, optional reference files, and optional asset files.

### What Does the Directory Structure Include?

The directory structure includes **5 files organized across 3 directories**:

- **SKILL.md**: 271 lines containing YAML frontmatter and condensed instructions for all 71 rules
- **references/content-optimization.md**: 341 lines covering Rules 1 through 20 with full examples
- **references/semantic-seo.md**: 455 lines covering Rules 21 through 45 with Google patent references
- **references/advanced-techniques.md**: 463 lines covering Rules 46 through 71 with vocabulary richness details
- **assets/checklist.md**: 85 lines providing a quick-reference checklist of all 71 rules

The total skill spans 1,615 lines across these 5 files.

### How Does Progressive Disclosure Work in Agent Skills?

Progressive disclosure works in Agent Skills by loading information in **3 stages** based on task requirements.

Stage 1 loads the metadata, approximately 100 tokens, containing the skill name and description. Stage 2 loads the full SKILL.md body, recommended to remain under 5,000 tokens, when the agent activates the skill. Stage 3 loads reference files and assets only when the agent requires detailed examples or checklists for a specific rule category.

This 3-stage approach prevents context window overflow. The main SKILL.md file remains under 500 lines, following the specification's recommendation. Detailed reference material resides in separate files that the agent loads on demand.

This skill can be deployed across AI platforms by copying the skill directory to the platform-specific skills location. There are 3 primary deployment paths:

1. **Copy to `.agents/skills/`** for VS Code with GitHub Copilot integration

The YAML frontmatter contains the skill name, description, author metadata, version number, and framework attribution. Any agent that supports the Agent Skills format discovers and activates the skill automatically when a content writing task matches the skill description.

### What YAML Frontmatter Fields Does the Skill Require?

The skill requires **5 frontmatter fields** in the SKILL.md file:

- **name**: lowercase, hyphenated identifier matching the directory name, maximum 64 characters
- **description**: 1 to 1,024 characters explaining the skill purpose and activation triggers
- **license**: license type or reference to a bundled license file
- **metadata**: key-value pairs for author, version, author URL, and framework attribution
- **framework**: the underlying methodology, in this case Koray Tugberk Gubur's Semantic SEO framework
