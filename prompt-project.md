## 2. Metadata Setup

### 2.1 SEO Title Tag
- 50-60 characters.
- Put the Primary Keyword (website name) at the very front.
- Use specific numeric values where applicable (Rule 66).

### 2.2 Meta Description
- Exactly 150-155 characters.
- Must include the Primary Keyword.
- Include a compelling, factual call to action.
- Use formal and professional diction only (Rule 8).

### 2.3 Overview (300-400 words)
- Start with a factual definition or direct statement (Rule 30).
- Naturally integrate the secondary keywords throughout this section.
- The Primary Keyword must appear wrapped in bold (**strong**) tags within the first 100 words.
- Include 2-3 authoritative source citations (Rule 16).
- Provide perspective richness where applicable (Rule 25, Rule 47).
- Do not break context across paragraphs. Keep related details within one paragraph (Rule 38).
- Follow a logical order in declarations. Each point must build on the previous one (Rule 39).

### 2.4 The Challenge & Solution (approx. 400 words)
- Scroll the website and detail the specific problem solved.
- Use the format: State the challenge first, then the solution approach, then the results.
- Expand evidence with variations: include statistical evidence, expert opinions, and specific examples (Rule 29).
- Match the tenses and modality of answers with the questions asked (Rule 41).
- Optimize the first sentence under each heading to match the heading format (Rule 45).
- Do not distance the question from the answer. Answer immediately, then elaborate (Rule 60).

### 2.5 Conclusion (approx. 300 words)
- Summarize the results with specific, verifiable statements (Rule 30).
- Include the same n-grams used in the overview to reinforce context (Rule 53).
- End with a context that mirrors the H1 topic for a complete context vector (Rule 22).
- Do not promote products in informational content (Rule 10).

## 3. General Writing Rules and Guidelines

All generated content in Section 2 must adhere to these mandatory rules:

### 3.1 Global Writing Rules (Apply to All Sections)

**Sentence Structure:**
- Use simple sentence structures: Subject + Verb + Object/Modifier (Rule 44).
- Use shorter sentences. Give one piece of information per sentence (Rule 68).
- Structure key sentences using semantic triples: Subject + Special Fact + Main Information (Rule 11, Rule 34).
- Do not create extra sentences without logical reason. Combine related information where possible (Rule 12).

**Language and Tone:**
- Use formal and professional diction only. No slang, colloquialisms, contractions, idioms, or phrasal verbs (Rule 8).
- Do not use analogies. Replace all comparisons with direct, factual statements (Rule 5).
- Be certain. Use definite, factual statements. Avoid possibility language (Rule 13).
- Be specific. Use exact counts and categories instead of vague descriptors like "many" or "various" (Rule 14, Rule 66).

**Entity and Context Rules:**
- Focus on entities and their attributes, not just keywords (Rule 27).
- Do not stuff entities. Prioritize by prominence, relatedness, and popularity (Rule 28).
- Include relevant entities and attribute values that fit the macro context (Rule 26).
- Key terms must connect to the title and H1, and in turn connect to the topical map (Rule 36).
- Maintain a single macro context per webpage. Do not introduce unrelated topics (Rule 57).

**Pronoun and Reference Rules:**
- Every pronoun must clearly refer to a specific entity. No coreference errors (Rule 7).
- Do not reference previous sections with "As stated before" or "As explained above." Restate information directly (Rule 6).

**Citation Rules:**
- Cite authoritative sources before giving statements. Use format: "According to [Source], [Statement]" (Rule 16).
- Mention studies using format: "A [Year] study at [Organization] found that [Fact]" (Rule 42).
- Do not add outbound links to citations. Mention sources in text only (Rule 9).

**Formatting Rules:**
- Use ordered lists for sequential steps and unordered lists for non-sequential items (Rule 67).
- Use the same part of speech at the start of every list item (Rule 19).
- Bold the answer part, not the search term (Rule 56).
- Give examples after every plural noun (Rule 33).
- When mentioning measurements, include multiple unit systems (Rule 65).

### 3.2 Heading Structure

- Use a strict H1-H6 semantic hierarchy. Do not skip heading levels.
- Create a proper context vector from H1 to the last heading (Rule 22).
- Convert headings to questions where applicable for better search engine matching (Rule 61).
- Include the Primary Keyword naturally in at least two subheadings.
- Use short-form questions for headings. Split long-form questions into H2 (grouper) and H3 (specific) (Rule 55).

### 3.3 Project Title (H1)

- The H1 must be the Website Name (Primary Keyword).
- This is the single most important on-page SEO element (Rule 36).
